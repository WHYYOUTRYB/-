# 笔记写作规则

> 本文件是添加/编辑笔记时的唯一规范参考，所有规则均由 `src/content.config.ts`、`src/content/subjects.config.ts`、`src/lib/notes.ts`、`astro.config.mjs` 实际约束。

---

## 1. 文件位置与命名

- 笔记统一放在 `src/content/notes/` 下，扩展名 `.md`。
- 可扁平存放，也可按学科建子目录（如 `notes/laser/ch1a-光与原子.md`）。子目录会进入笔记 `id`，进而影响 URL（见第 3 节）。
- 文件名建议格式 `YYYY-MM-DD-slug.md` 或 `chN-标题.md`，仅作可读性约定，不参与排序。

---

## 2. frontmatter 字段

```yaml
---
title: 极限的定义              # 必填，笔记标题
subject: advanced-math        # 必填，学科 slug，必须在 subjects.config.ts 中已定义
chapter: 第一章 极限与连续      # 必填，章节名（自由文本），学科页据此动态分组
order: 1                      # 可选，章节内排序；同时决定该章节在学科页的排序权重
pubDate: 2026-03-01           # 必填，发布日期（YYYY-MM-DD）
description: 一句话描述         # 必填，列表页/SEO 用
tags: [极限, 定义]             # 可选，标签数组，缺省为 []
draft: false                  # 可选，true 时不生成页面、不进列表，缺省 false
---
```

### 字段约束（Zod schema 强校验）

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | ✅ | — |
| `subject` | enum | ✅ | 取值必须是 `subjects.config.ts` 里某个 `slug`，否则构建报错 |
| `chapter` | string | ✅ | 同一章的笔记 `chapter` 文本必须**完全一致**（含空格/标点），否则会被分成两组 |
| `order` | number | — | 越小越靠前；缺省按 `pubDate` 倒序 |
| `pubDate` | date | ✅ | 字符串会被自动解析为日期 |
| `description` | string | ✅ | — |
| `tags` | string[] | — | 缺省 `[]` |
| `draft` | boolean | — | 缺省 `false` |

> 字段值含特殊字符（如 `:`、`#`、`—`）时，用引号包起来，例如 `title: "第0章 前言 — 什么是激光"`。

---

## 3. URL 规则

```
/notes/<subject>/<note.id>
```

- `<subject>` = frontmatter 的 `subject`。
- `<note.id>` = 文件相对于 `notes/` 的路径（去掉 `.md`）。子目录会保留在 URL 中。

| 文件 | subject | URL |
|------|---------|-----|
| `notes/2026-03-01-limit-definition.md` | `advanced-math` | `/notes/advanced-math/2026-03-01-limit-definition` |
| `notes/laser/ch1a-光与原子.md` | `laser-physics` | `/notes/laser-physics/laser/ch1a-光与原子` |

> URL 含中文是正常的（浏览器自动 percent-encode）。`note.id` 一旦确定就不宜改动，会断链。

---

## 4. 排序规则

### 学科页（`/notes/[subject]`）按章节分组

- **章节内**：`order` 升序；无 `order` 的按 `pubDate` 倒序。
- **章节间**：按该章节**首篇笔记的 `order`** 排序（若无 `order`，按首次出现顺序）。

⚠️ 这意味着：给某章第一篇笔记设 `order`，等于给整章定位。建议同一学科内所有笔记都显式写 `order`，章节顺序才稳定可控。

### 首页「最近笔记」与笔记总览

按 `pubDate` 倒序，取最近 N 篇。

### 笔记详情页「上下篇」

同学科内，先按 `chapter` 字符串排序，再按 `order`、再按 `pubDate`。

## 5. Markdown 扩展

### 数学公式（KaTeX）

- **行内**：`$E = mc^2$` —— 单 `$` 必须写在**同一行内**，不可跨行。
- **块级**：`$$` 单独成行包裹，前后各空一行：

  ```markdown
  $$
  \lim_{n \to \infty} a_n = A
  $$
  ```

⚠️ **常见坑**：块级公式误用单独一行的单 `$`（即 `$` 换行、公式、再 `$` 换行）**不会被渲染**——remark-math 的行内数学不允许跨行。块级必须用 `$$`。历史上激光 ch1a 因此出过问题，已修复。

### 代码高亮（Shiki 双主题）

用标准围栏 ` ``` `，浅色 github-light / 深色 github-dark-dimmed 自动随主题切换：

````
```python
def f(x):
    return x ** 2
```
````

### Wiki 链接

`[[笔记标题]]` 会被解析为站内搜索链接 `/search?q=<标题>`，不依赖 slug，标题改了也不会断链。

### 图片

放 `public/images/<学科>/`，用绝对路径引用：

```markdown
![说明](/images/laser/fig1_lineshape_compare.png)
```

### 交互演示（HTML）

放 `public/demos/<学科>/`，用绝对路径链接：

```markdown
[行波三重视角演示](/demos/laser/day2_行波_三重视角.html)
```

> 引用前确认文件真实存在，否则 404。

## 6. 学科管理

学科不写死，由 `src/content/subjects.config.ts` 驱动。**加学科 = 加一行配置**：

```ts
{
  slug: "new-subject",        // URL 标识，唯一，确定后不宜改
  name: "新学科",              // 显示名
  description: "一句话介绍",   // 学科页副标题
  icon: "◇",                  // 极简风格用字符代替图标
  order: 5,                   // 学科卡片排序，越小越靠前
},
```

- `slug` 一经确定不要改，会影响所有该学科笔记的 URL。
- 笔记通过 frontmatter 的 `subject` 字段（值 = slug）归属学科。
- 删学科只需删配置项，不影响 `notes/` 里的笔记文件本身（但那些笔记会因 `subject` 校验失败而构建报错，需同步删除或迁移）。

---

## 7. 草稿

`draft: true` 的笔记：
- 不生成详情页
- 不出现在任何列表（首页、学科页、标签页、搜索索引、RSS、sitemap）

适合写到一半先存档。发布时改回 `false`。
